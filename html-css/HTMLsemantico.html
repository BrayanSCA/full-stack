Default distribution web page

<header>

La cabecera es la parte superior de la página web. Aquí suele incluirse la información básica de la empresa o marca y es consistente en todo el sitio, es decir, se repite en cada página de la web por la que navegas.

<nav>

La navegación es la parte de la página web que contiene los enlaces a otras páginas del sitio. En este caso, el menú de navegación incluye enlaces a las secciones de registro, información, instructores y testimonios.

<article>

El artículo es una sección independiente de contenido que puede ser distribuido y reutilizado. En este caso, el artículo incluye información sobre el taller de Machine Learning.

<section>

La sección es una parte de la página web que contiene un conjunto relacionado de contenido. En este caso, la sección incluye información sobre el registro, los instructores y los testimonios.

<aside>

El aside es una sección de contenido que está relacionada con el contenido principal, pero no es parte de él. En este caso, el aside incluye información adicional sobre el taller de Machine Learning.

<footer>

El pie de página es la parte inferior de la página web. Aquí suele incluirse información adicional sobre la empresa o marca, como la dirección, el número de teléfono y los enlaces a las redes sociales.

<address>

El address es una sección de contenido que contiene información de contacto. En este caso, el address incluye la dirección de la empresa y el número de teléfono.

<div>

El div es un contenedor genérico que se utiliza para agrupar contenido relacionado. En este caso, el div se utiliza para agrupar la cabecera, la navegación y el pie de página.

<span>

El span es un contenedor en línea que se utiliza para agrupar contenido relacionado. En este caso, el span se utiliza para agrupar el texto dentro de un párrafo.

<figure>

El figure es una sección de contenido que contiene una imagen o gráfico. En este caso, el figure incluye una imagen del taller de Machine Learning.

<figcaption>

El figcaption es una sección de contenido que contiene una descripción de la imagen o gráfico. En este caso, el figcaption incluye una descripción del taller de Machine Learning.

<time>

El time es una sección de contenido que contiene información sobre la fecha y hora. En este caso, el time incluye la fecha y hora del taller de Machine Learning.

<mark>

El mark es una sección de contenido que contiene información resaltada. En este caso, el mark incluye información resaltada sobre el taller de Machine Learning.

<progress>

El progress es una sección de contenido que contiene información sobre el progreso. En este caso, el progress incluye información sobre el progreso del taller de Machine Learning.

<meter>

El meter es una sección de contenido que contiene información sobre la medida. En este caso, el meter incluye información sobre la medida del taller de Machine Learning.

<template>

El template es una sección de contenido que contiene un modelo o plantilla. En este caso, el template incluye un modelo o plantilla del taller de Machine Learning.

<summary>

El summary es una sección de contenido que contiene un resumen. En este caso, el summary incluye un resumen del taller de Machine Learning.

<details>

El details es una sección de contenido que contiene información adicional. En este caso, el details incluye información adicional sobre el taller de Machine Learning.

Un diseño con un encabezado, dos barras laterales y un pie de página se conoce como el diseño sagrado grial. Existen muchas formas de marcar este contenido, como las siguientes:

    
      < body>
        < header>Header < /header>
        < nav>Nav < /nav>
        < main>Content < /main>
        < aside>Aside < /aside>
        < footer>Footer < /footer>
      < /body>

Si estás creando un blog, es posible que tengas una serie de artículos en < main>:

    
      <body>
      <header>Header </header>
      <nav>Nav </nav>
      <main>
        <article>First post </article>
        <article>Second post </article>
      </main>
      <aside>Aside </aside>
      <footer>Footer </footer>
      </body>

diferencias entre <article> y <section>

<article> : es como un capítulo completo o una publicación individual. <section> : es como una parte o una sección dentro de ese capítulo o publicación, o dentro de la página principal.

Atributes

Los atributos son propiedades que se pueden agregar a las etiquetas HTML para proporcionar información adicional sobre el contenido. Algunos atributos comunes son:

id=""

para la etiqueta <label> se usa el atributo "for" para asociarlo a una etiqueta <input> sin embargo este puede ser omitido si el input se encuentra anidado dentro de label. Ejemplos:

input sin anidar

      
        <label for="nombre">Nombre:</label>
        <input type="text" id="nombre">

input anidado en label

      
        <label>
          Nombre:
          <input type="text">
        </label>

Los atributos "id" distinguen mayúsculas de minúsculas por eso se debe tener cuidado al utilzarlos para poder ser llamados correctamente entre código HTML, CSS o JavaScript.

aria-lebelledby=""

El atributo "aria-labelledby" se utiliza para asociar un elemento con otro que actúa como su etiqueta. Esto es útil para mejorar la accesibilidad de la página web, especialmente para los lectores de pantalla. Por ejemplo:

      
        <label id="nombre-label">Nombre:</label>
        <input type="text" aria-labelledby="nombre-label">

En este caso, el atributo "aria-labelledby" asocia el input con la etiqueta "Nombre:". Esto significa que cuando un lector de pantalla lee el input, también leerá la etiqueta asociada.

Hay más de 50 estados y propiedades de aria-* que se pueden usar para garantizar la accesibilidad. aria-labelledby, aria-describedby, aria-details y aria-owns toman como valor una lista de referencias id separada por espacios. aria-activedescendant, que identifica el elemento descendiente enfocado actualmente, toma como valor una sola referencia id: la del elemento único que tiene el enfoque (solo se puede enfocar un elemento a la vez).
